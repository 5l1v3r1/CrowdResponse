CrowdResponse - 1.0.6.0

4/7/2016

  • Added - new module @tasks
    • We now have tasksv1 and tasksv2 output.
      • tasksv1 uses the original Task Scheduler 1.0 API.
      • tasksv2 uses the newer (Vista+) Task Scheduler 1.2 ("2.0") API.
  • Added - new module @jobs to enumerate 'at' jobs.
  • Added - introduction of the concept of fileops for module file operations.
    • Many modules had several options to control various operations on discovered files such as reporting file version, certificate validation, hashes etc. These were all separate command line options. We have now replaced these disparate options with a single option (typically -t) followed by a string that can provide all of the previous options in one place. See the readme file for full details.
    • This obviously requires that previous configuration files be updated to support the new format.
  • Added - new application variable %hdnext%.
    • This corresponds to the next physical hard drive (i.e. not network or removable drives) after %systemdrive%.
    • If there is no next drive it will translate to "?:"
    • For example "%hdnext%\MyFiles" could translate to "D:\MyFiles"
    • In addition to the places where the other pseudo environment variables are used, this new one can also be used in @dirlist and @mft.
  • Added - new application variable %hdprev%.
    • Similar to %hdnext% above, this variable translates to the hard drive prior to %systemdrive%.
  • Added - new application variable %dm% that resolves to the Windows domain name.
  • Added - Windows domain name to <system> output section, "domain".
  • Added - index Id value to entries in @shim.
  • Added - module run duration to XML log.
  • Added - new @psstrings option -C (note capital C) to specify a custom character set.
  • Added - option -l (el) to @psstrings for credit card Luhn check validation.
  • Added - module execution order list to readme file.
  • Added - application start time, end time, duration and last HTTP upload status code to the XML log section regardless of @main option -d (Log basic application information to the registry).
  • Added - DisplaySwitch application to @mal -s detection.
  • Added - ability to enumerate processes by name in @psstrings.
    • This is achieved through the use of new option -P <name> where name is a regular expression applied to the full process file path.
  • Added - WinHttpAutoProxySvc service dump to @sensor.
  • Added - Schannel Protocols registry dump output to @sensor.
  • Added - alternate CS Agent registry information location output to @sensor.
    • This did not affect operation of the application. It was just potentially confusing for the user.
  • Added - @main option -q to test for free disk space.
    • If there is insufficient free disk space on the output volume prior to starting the application will exit.
  • Added - new log XML section to record warnings and errors encountered during the running of the application.
  • Added - process priority value and priority string to @pslist.
  • Added - session id value to @pslist.
  • Added parent PID value to @pslist.
  • Added - @main option -g to validate and report on certain issues that can occur in a configuration file that can result in problematic XML output. e.g. incorrect text encoding (Unicode characters present), misplaced double quotes etc.
  • Added - option -r to @regdump to enable connection to remote registries.
  • Added - Most places where a system environment variable is supported we now also support built-in application environment variables (%hdnext% etc).
  • Added - new options for @regdump to output additional entries corresponding to interpreted binary data in text format.
    • Option -t will output the binary data interpreted as single byte ASCII characters.
      • The corresponding pseudo value type in the XML output is binary_textA.
    • Option -T will output the binary data interpreted as double byte (wide character) Unicode characters.
      • The corresponding pseudo value type in the XML output is binary_textW.
    • Unprintable characters are shown as a period '.'.
  • Added - new module: @system.
    • This is basically a no-op in that it does not add any output of its own but allows for the application to simply emit the already present <system>...</system> information without having to invoke any other module. This can be useful as a basic inventory.
  • Added - new @main option -s to rate limit HTTP uploads.
  • Added - @main option -u now supports file directories as the destination for "upload".
    • This is useful in situations where the final output destination path is on a file share but we don't want to incur all of the overhead of incrementally writing to the file share and compressing the file on the share during the running of the application.
  • Added - @main option -Y for proxy fall-back.
    • When uploading the resultant file (-u, to a URL) if proxy upload fails (-y) revert to no proxy and try again.
  • Fixed - bug passing on command line parameters from main stub application to platform specific version.
  • Fixed - improperly formed XML output of base address in @drivers.
  • Fixed - the application now prevents system dialog boxes from showing when attempting to access devices that have removable media not present.
  • Fixed - the embedded 32 bit version of the application was not being digitally signed in the official build process. This has been remedied.
  • Fixed - in the @mal module in the case where the referenced file for a debugger for the associated accessibility app does not have a full path specified (e.g. Just specifying "cmd.exe") then we would not be showing the details of the file (hashes, properties etc). It would still show as vulnerable but there would just be no file details. We will now assume that the full path of a file with no drive/directory present refers to the Windows system directory.
  • Fixed bug in @mal -s.
    • If the file in question did not exist on disk it would be reported as vulnerable.
  • Fixed a problem with indentation of XML tags.
  • Fixed a bug in @yara when using multiple YARA sections in a config file leading to incorrect processing of rules.
  • Fixed problem with truncation of regular expressions in @dirlist and other places where environment variables are expanded.
  • Fixed bug in @pslist when using -x option (For -e, don't dump full resource information)
  • Fixed issue with catalog based digital signatures on Windows 8 and above.
  • Changed - Module @psstrings option -i ("Only include strings matching regex") is now option -g.
  • Removed - the seldom used @psstrings option -m ("Text containing characters that MUST be present in a string"). This is effectively replaced by option -C (see below).
  • Improved - detection of binary characters in configuration file.
  • Improved - output of offending area of configuration file when errors are detected.
  • Improved - URL for @main -u (upload to server) now supports the built-in pseudo environment variables (%cu%, %ag% etc).
  • Improved - @main option -d (Log basic application information to the registry) now also logs the last HTTP file upload server status code to the registry.
  • Improved - @main option -g (Validate config file and exit) now also checks the configuration's parameters syntax.
  • Improved - in all cases where a path inclusion/exclusion regex is supplied, the regex will now have all single character back slash characters escaped to become 2 back slashes.
  • Improved reporting of improperly constructed configuration file SECTION names.
  • Improved - reduced superfluous output from @yara module when using option -h (Only show positive hits).
  • Improved @main option -g (validate config) to detect config file issues at an earlier stage (prior to extracting YARA sections for example).
    • If this option is supplied the application will check for issues and exit immediately. No further actions will take place.
  • Updated - new features for @mal -d (DLL hijacking).
    • There are now 3 distinct tests for this option:
      1. Performs checking of DLL file name conflicts within each directory found in the environment variable PATH. i.e looks for the same base DLL file name being present inside 2 or more PATH directories.
      2. Performs checking of DLL file name conflicts for files not found in a PATH directory against those found in a PATH directory.
        • Directory values listed at SYSTEM\CurrentControlSet\Control\Session Manager\KnownDLLs (DllDirectory and DllDirectory32) will be ignored.
      3. Examines all svchost.exe processes (Windows services) that are running and for each one analyzes the list of modules that the process has loaded. If any module of the same base file name (DLL) is loaded at a different path than another a conflict is reported.
  • Updated - The @dirlist module will now process actual directory entries.
    • The <path> XML element will point to the path of the directory.
    • The <name> field will be a blank string.
    • The <attr> "str" attribute will contain "DIRECTORY".
    • Note that several file operations are not really applicable to directory entries and will be blank or show an error message.
  • Updated - @mal -a sticky Sticky Keys list of allowed valid certificate signer strings to accommodate values found on Windows Server 2003.
  • Updated - additional protocols for HTTPS uploads.
    • By default only SSL3 and TLS1 were enabled in Windows 7 and Windows 8.
    • By default only SSL3, TLS1.0, TLS1.1, and TLS1.2 were enabled in Windows 8.1 and Windows 10.
    • SSL3, TLS1.0, TLS1.1, and TLS1.2 are now explicitly enabled on all platforms.
    • This would only have an effect if the web server for example only accepted TLS1.2.
    • HKLM\SOFTWARE\CrowdStrike\CrowdResponse UploadStatus (DWORD)

CrowdResponse - 1.0.4.0

12/19/2014

  • New modules @prefetch, @shim and @superfetch.
  • @pslist option -t (check for thread injection) is now option -i.
  • @pslist option -j (do MD5 hashes) is now option -m.
  • @pslist option -m (enumerate loaded modules for each process) is now option -o
  • @pslist option -r (don't dump full resource information) is now option -x
  • @pslist added option -z (don't hash files larger than <size>)
  • Removed redundant XML path tool_PSList/processes/process/peinfo/file/name for @pslist.
  • Several new options added to @regfile module, replacing the hard coded features previously in place
    • -c Verify digital signature of service modules
    • -h Do SHA256 hashes
    • -m Do MD5 hashes
    • -q Perform "quick" hash of only the 1st 512 bytes of the file
    • -r Display application resource information
    • -t Display file create, modified and accessed times
    • -z <size> Don't hash files greater than this size in megabytes
  • Recommended options for @regfile should now then be @regfile -c -h -m -r -t
  • Certificate checking in all modules now also checks the Windows Security catalogs for signing along with the original tests for embedded signatures. Because of this there is now a <cert_type> XML field emitted and present in the converted files in CRConvert.
  • Certificate checking should now no longer reach out to the network when performing revocation checks. This is a speed optimization. Validation is now only performed using information cached by the operating system.
  • New @yara option -# <section> that can be used to specify a particular built-in config [<SECTION:yara>] section, indexed from zero. Using this feature you can now use the same config file but reference different YARA script sections within that config file for each @yara instance.
  • Fixed bug in @superfetch where there was a missing XML container element ("appentries") when run on certain platforms.
  • Several new options added to @drivers module, replacing the hard coded features previously in place
    • -c Verify digital signature of service modules
    • -h Do SHA256 hashes
    • -m Do MD5 hashes
    • -q Perform "quick" hash of only the 1st 512 bytes of the file
    • -r Display application resource information
    • -t Display file create, modified and accessed times
    • -z <size> Don't hash files greater than this size in megabytes
  • Recommended options for @drivers should now then be @drivers -c -h -m -r -t
  • Added new field lastRegWriteTime to @regdump and @regfile output for the parent key, representing the last time (UTC) the key was written to.
  • Fixed behavior of @dirlist -z option ("Don't hash files greater than this size in megabytes") such that if a file is above this limit we still emit the other details for the file instead of skipping the file altogether.
  • If @dirlist option -z as described above is used, the file hash values will be shown as [ignored] in the event that a file is larger than the specified value.
  • In the event that a file cannot be accessed in @dirlist, the file hash values will show as [error]
  • Reverted change for @dirlist file attributes output. We now again emit 0xffffffff if the file cannot be accessed. In addition the XML str value for the file attribute will show [error] in this case.
  • Added isAdmin field to System XML section. This signifies whether the application was launched with admin privileges or not.
  • Added option -v to @pslist to enumerate DLLs used by svchost.exe processes. Note that this option is redundant and will be ignored when used in conjunction with option -m (enumerate loaded modules) since loaded modules would include the same DLLs. Option -v is only really useful if you only want to see what DLLs a svchost.exe process is utilizing.
  • Improved responsiveness to HTTP errors when uploading.
  • @prefetch - New option -t to translate paths. See below.
  • @superfetch - New option -t to translate paths. See below.
  • When reading a config file all carriage-return linefeed line endings (0x0d [\r] 0x0a [\n]) are now replaced with linefeed characters (0x0a [\n]). This is a workaround for a bug/feature in the YARA library when processing hex strings where it was not including carriage-return characters as white space.
  • Significant reduction in memory usage for @YARA module when scanning process memory. This is a change to the actual YARA library code.
    • Instead of creating a copy of all allocated memory regions for the process and then applying rules to that large duplicated collection in a batch, I now only copy and apply rules to each individual region one at a time.
    • The original mechanism effectively consumed twice the amount of memory that the process was using. This new method will only duplicate the largest region size, which is almost always significantly smaller than the entire process' memory.
    • In addition I utilize a single allocated buffer to store the copied memory region, releasing the memory and dynamically increasing its size only when a larger region is encountered. This reduces memory allocation/release thrashing.

File paths that start with e.g. "\device\harddiskvolumeX\dir\file.exe" are now translated into their logical drive paths e.g. "C:\dir\file.exe". Prefetch paths for example are now shown in this manner.

CrowdResponse - 1.0.3.0

8/19/2014

  • New module @RegDump for dumping registry items.
  • New module @RegFile for finding and reporting of registry entries referring to files.
  • Added support for web proxy servers using -y.
  • @main option -i (load config from file) can now also specify a URL as the target to download the config file from the provided location.
  • New pseudo environment variable %ad% that represents the application's directory. It expands to a directory path including the trailing back-slash '\'.
  • New pseudo environment variable %ve% for use in output file naming. This will emit the application product version e.g. "1.0.3.0".
  • You can use pseudo environment variables for the input config file now, although only the new %ad% really makes sense. This latter case can be useful when running the tool as another user (RunAs) when the current directory often ends up being something like "C:\Windows\System32" thus preventing you from locating files that were not referenced by their full path.
  • Changed file open sharing permissions on the output XML file to prevent multiple instances from writing to the same file.
  • Fix for self destruct (-x) not working when EXE is run from a different volume than where the system's temporary directory resides.
  • Revamped and simplified the self-destruct code (main option -x) to make it far more robust. It was discovered that previously a copy of the application could occasionally be found in the system's temp directory.
  • Accommodated situations in modules such as @dirlist where we could be tricked into reporting incorrect file meta-data information in certain cases. An example is if a file name is created with a trailing space character. Most regular Windows API functions will strip away the space prior to processing the name and thus potentially then refer to a different file name. The solution is to use extended file references (adding '\\?\' to the start of the name) to force the use of the raw name. Note that even then not all Windows API functions operate correctly, notably the GetFileVersion() function used to obtain version information from a file that will still be fooled by the example given.
  • If an input config file is specified without a full path and it is not found in the current directory when the application starts it will attempt to locate it in the application's directory.
  • Both the input config file path (-i) and output file path (-o) can now also utilize regular system environment variables.
  • Additional output is shown in the event the application executable file(s) cannot be deleted when the self-destruct (-x) option is used.
