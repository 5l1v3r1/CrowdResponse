CRConvert - Version 1.0.5.3

5/13/2015

  • Support for new module @sensor.
  • Added option -d to supply a path to a SQLite3 database file containing a whitelist of SHA256 hashes.
    • This database is queried when processing @dirlist output and produces 2 additional output files, one containing details of files matched against the whitelist and the other details on files that do not match the whitelist.
    • This option requires the sqlite3.dll file to be present in your path.
    • This can be downloaded from https://www.sqlite.org/download.html
  • Added option -z to ignore zero byte sized files when using the whitelist feature (-d).
  • Added option -b to specify the SQLite table name for the -d database.
  • Added option -n to specify the column name for the -d database.
  • If a provided XML file is unable to be processed due to its size we will now attempt to internally split the file by each referenced module (tool name) instance and process those individually.
    • By doing this we may be able to process most if not all of the file.
    • Only if any of the split out individual module instances are themselves too large will that data not make it to the resultant output.

CRConvert - Version 1.0.5.0

4/17/2015

  • New module @mal.
  • Complete rewrite of core XML processing producing a dramatic decrease in processing time.
  • Added option -x to output in plain text format, one field per line.
  • Added missing OSInstall field for system
  • Fixed problem with HTML output always being appended even without using -a.
  • For HTML output (-h) empty columns are now emitted as a non-breaking space character.
  • Fixed missing ampersand '&' character in decimal entities in output.
  • Fixed missing digital certificate type (embedded, catalog) in some module output.
  • Improved handling of append mode for HTML output.
  • When creating the output directory all intermediate directories specified in the path will be created if they don't exist. Previously only the first level directory would have been created.
