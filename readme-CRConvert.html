CRconvert

CRconvert is a companion tool to CrowdResponse and is used to convert the native output of the former into other formats for easier human readability and more straightforward ingestion into automated data processing applications.

Overview

By default, output from CrowdResponse is provided in an XML file. CRconvert will flatten this XML to CSV (comma separated values), TSV (tabbed separated values) HTML or plain Text, if desired. The HTML output may be viewed in any browser. The CSV and TSV output may be processed via a spreadsheet application of your choice, such as Microsoft Excel or OpenOffice Calc, or via a data analysis platform such as Splunk. The various format options were created to support the different needs and analysis preferences of the end user.

Usage

This tool provides users with an opportunity to easily create CSV, TSV or HTML data from the standard XML output of CrowdResponse.

This is the current syntax for the tool as seen when simply typing CRconvert at the command prompt:

CRconvert
  CrowdResponse XML to CSV, HTML and Text converter
  Version 1.0.6.0 (32 bit)
  Copyright (C) CrowdStrike, Inc. 2016.  All rights reserved.
  http://www.crowdstrike.com

Syntax:

CRconvert -f <path/mask>
          [-b <table>]
          [-d <database>]
          [-e <prefix>]
          [-m <rows>]
          [-n <name>]
          [-o <dir>]
          [-achprstvx]

  -a             Append output files (default is overwrite)
  -b <table>     Database table name for option -d. Default is "Hashes"
  -c             Output in CSV format
  -d <database>  Path to SQLite3 database of whitelisted SHA256 hashes
  -e <prefix>    Output file name prefix. Default is "CrowdResponse_"
  -f <path/mask> File name path/mask. Default mask is "*.xml"
  -h             Output in HTML format
  -m <rows>      Limit rows per CSV output file creating new ones when reached
  -n <name>      Database SHA256 column name for option -d. Default is "SHA256"
  -o <dir>       Output directory for files. Default is current
  -p             Use low CPU priority (idle) to lessen load on the system
  -r             Recursively search input directory when looking for files
  -s             Do not use XML file name as first field ("system")
  -t             Use tabs to separate CSV output fields instead of commas
  -v             Verbose output
  -x             Output in text format
  -z             Ignore zero byte sized file when using whitelist (option -d)

The filename "stdin" can be used with -f to specify input from stdin.

Option -f can be used both as a wildcard filter for the file name component
of paths and to specify the directory for input XML files. For example

  -f myfile.xml

would process the single file myfile.xml in the current directory

  -f C:\MyXMLFiles\myfile.xml

would process the single file myfile.xml in the directory C:\MyXMLFiles

  -f C:\MyXMLFiles

would process all *.xml files in the directory C:\MyXMLFiles

  -f C:\MyXMLFiles\test*.xml

would process all test*.xml files in the directory C:\MyXMLFiles

In the absence of -c (CSV), -h (HTML) or -x (Text) the default will be -c.

-c, -h and -x can be specified together to output in any/all formats.

Row limiting mode (-m) for CSV output will create files of the form
'mod_system_00000.csv' with the number incrementing for each file. Existing
csv files that were produced without using -m will not be considered when
appending (option -a).

Option -d specifies a SQLite database file containing SHA256 hashes of files
considered to be whitelisted and are cross referenced with the hashes found
during @dirlist processing. The hashes should be in a table specified using
option -b in the field named with option -n. Hashes should be lowercase.
Note that the file "sqlite3.dll" must be in your path to enable this feature.

Examples:

  CRconvert -v -f MyXMLDir -o MyCSVDir
  CRconvert -v -c -h -f MyFile.xml

Supported XML sections/modules:

  log                      dirlist                  dirlist_not_whitelist
  dirlist_whitelist        drivers                  handles
  jobs                     mal_dllhijacking_inpath  mal_dllhijacking_outpath
  mal_dllhijacking_svchost mal_stickykeys           prefetch
  pslist_modules           pslist                   psstrings
  regdump                  regfile                  sensor_lookup
  sensor_main              shim                     superfetch_appentries
  superfetch_perioddata    system                   tasks
  tasksv1                  tasksv2                  yara

Upon conversion of the XML data via CRconvert, you’ll have a CSV, TSV or HTML file for each sub-tool module selected, in addition to one for overall main system information collected during execution. The system information provides an inventory of the systems that have been scanned along with associated data point.

Note: CRconvert may be run in a Linux environment using Wine.

Note: Some of the modules in CrowdResponse contain data that is unsuitable for conversion into a tabular format. An example would be the individual modules associated with a loaded process, of which there can be a considerable and variable number. Therefore the decision was made to not include such variable fields in the converted output.
